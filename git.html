GIT

Git est un logiciel de versionnement de code qui fonctionne avec un système d'arborescence à partir d'une branche master. Git fait constamment le lien entre un repository local et un repository hébergé par GitHub. Un repository est un dossier qui est configuré pour travailler avec git.

GitHub est un serveur qui permet de créer et stocker des repositories en utilisant les fonctionnalités de git.

Configuration

Pour connaître la version du logiciel git : 

git --version

Pour configurer les données utilisateur de git :

git config --global user.name "monNom"
git config --global user-email "monEmail"

Il faut créer un dossier local qui va héberger les différents repositories, par exemple Git_Directory. Puis se placer dans Git_Directory et initialiser le dossier :

git init

Il faut ensuite importer un repository du master online :

git clone https://github/monAdresse

Principales instructions

Pour ajouter un fichier local au repository online : 

git add monFichier

Pour ajouter rapidement au repository online sans distinction de fichier : 

git add .

On travaille normalement dans le repository local. Pour synchroniser le repository online master avec le repository local origin, on commence par rajouter dans l'index chaque fichier modifié, on demande la validation de cette modification et finalement on insère le fichier validé vers le repository online :

git add .
git commit -m "monCommentaire"
git push origin master

Il n'est pas nécessaire de définir le nom des fichiers concernés car tous les fichiers indexés et non validés vont être pris en compte par le commit. Attention, chaque étape est nécessaire. On peut toutefois regrouper les étapes du rajout à l'index et du commit :

git commit -a -m "monCommentaire"
git push origin master

Attention, les fichiers modifiés mais non entrés dans l'index ne seront pas synchronisés.

Pour connaître l'état des modifications :

git status

Pour récupérer un fichier de travail :

git pull monFichier master

Pour consulter l'historique :

git log

On peut se positionner à un moment de l'historique en ne tenant compte que des commit enregistrés jusqu'à un commit défini. On utilise alors git checkout codeCommit oû codeCommit est le code sha présent dans le log et représentatif d'un unique commit. A partir de cette requête, git log n'affiche que les commit enregistrés jusqu'au commit codeCommit. Pour revenir à l'état du dernier commit, on utilise l'instruction git checkout master.
